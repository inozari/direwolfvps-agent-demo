# User-friendly VPS checklist (for privacy and self-hosting)

“User-friendly” for a VPS is not a pastel UI. It is how fast a normal person can go from “I need a server” to “I have root and a clean IP” without a support ticket.

This checklist is what we optimize for at **DireWolfVPS** (online since **2018**, plans from about **$3.50/mo**).

## Before you pay

- [ ] Plans list CPU / RAM / NVMe / traffic in one table  
- [ ] Location pricing is visible (no surprise multipliers after checkout)  
- [ ] Payment methods are clear (we are crypto-only: BTC, XMR, ETH, USDT, LTC, …)  
- [ ] No KYC / no ID selfie required  

## At checkout

- [ ] One email or token is enough to receive access  
- [ ] Invoice settles on-chain — no card, no bank trail  
- [ ] You know what happens if payment is slow (invoice stays open until confirmations)

## After deploy

- [ ] Root SSH works in about a minute  
- [ ] Reinstall is one click  
- [ ] IPv4 is screened (Spamhaus / common blocklists)  
- [ ] IPv6 `/64` is included  
- [ ] DDoS filtering is on by default  

## Why this matters for self-hosting

Self-hosters usually want:

1. **Low monthly cost** so a side project can live for years  
2. **Predictable ops** so you are not fighting the panel  
3. **A host that is still there next year**
